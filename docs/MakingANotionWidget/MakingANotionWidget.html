Making a Notion Widget

home

Notion is a note-taking and productivity software that was launched in 2016. It has an interesting feature of adding your own widgets to personalise your notes. There is a wide range of widgets that Notion provides by default. However, there is also a large amount contributed by the notion user community. In this article I attempt to make my own notion widget.

Introduction

I aim to make a digital clock widget for notion. For this a very basic knowledge of HTML, CSS, and Javascript is necessary. However, the more the better.

Writing code

First we focus on making the widget without worrying about adding it to Notion. Assume it to be an independent HTML page.

Finding a Colour Scheme

Since Notion is known for making aesthetically pleasing notes, we will try to make our widget pleasant by following a colour palette. I use https://colorhunt.co/ for finding colour palettes.

I choose this colour palette for the clock.

Making a Static Clock

First we just make the structure of the clock with the required styling. This will not contain any working parts and it will be static.

I create a file called digitalClock.html which will have the HTML part of the widget. Initially it will have the following code: -

We observe, that the hour, minute and second dial are one after the other vertically. However, we want them to be one after the other in the same line, i.e., horizontally. We can use Flex to arrange these divs accordingly.

#clock {
    border: 1px solid black;
    width: fit-content;
    height: fit-content;
    display: flex; /* The clock container is now a flex-box */
    flex-direction: row; /* All the containers inside clock will be arranged in a row */
}

The gap attribute adds spacing between each element inside the flex-box container.

    border: 1px solid black;

    height: fit-content;
    display: flex; /* The clock container is now a flex-box */
    flex-direction: row; /* All the containers inside clock will be arranged in a row */

Now we will add colour to the widget.

First we add some variables that will store our colour’s hex-codes so that we can reuse them by just mentioning their variable names.

:root {
    --white: #EAEAEA;
    --teal: #00ABB3;
    --grey: #B2B2B2;
    --black: #3C4048;
}

Here we define four variables: white, teal, grey and black that are available to be used anywhere in the css file.

Now we can use these variables to add colour to the widget.

First, the hour, minute and second dials’ font colours and background colours are set to teal and black respectively using the variables we defined previously.

#hour-dial, #min-dial, #sec-dial {
    background-color: var(--black);
    color: var(--teal);
}

Since there is no text in the dials yet, we cannot see the font colour yet.

Get the Clock Ticking

Now we add the text to the dials. For this we need to use javascript because the text in the clock has to change as the time changes so static HTML is not enough.

For the javascript portion of code I again made a separate javascript file and added it to the html.

Below is the code I wrote in the Javascript file digitalClock.js: -

function updateTime() {
    var now = new Date(); //now becomes an object of Date class
		//The below three lines create references to the dials we made in the html file.
    const hourDiv = document.getElementById("hour-dial"); 
    const minDiv = document.getElementById("min-dial");
    const secDiv = document.getElementById("sec-dial");
		//The belo three lines retrieve the value of the current hour, minute and second value into three variables.
    currHour = now.getHours()
    currMin = now.getMinutes()
    currSec = now.getSeconds()
		// set the content of the element with the ID time to the formatted string
    // also 0 padding is added to the numbers for single digit time values
    hourDiv.innerHTML = currHour.toString().padStart(2, '0');
    minDiv.innerHTML = currMin.toString().padStart(2, '0');
    secDiv.innerHTML = currSec.toString().padStart(2, '0');
    //Since time changes every second, we make sure this update function is called every second.
    setTimeout(updateTime, 1000);
}
updateTime(); // this function is called as soon as the page is rendered (when script tag is rendered)

Making the Widget

To make the widget, we need to first host our html page. This is because, Notion widgets are just like embeddable links. However, only hosted html pages are embeddable. Also, a lot of hosting options are either complicated or expensive and may not be worth using for a project of this small size. So, I have found two simpler ways to host the page solely for the purpose of making this widget.

Method 1: Using Github Pages

Github allows you to host websites directly from you repository through Github Pages. This means the pre-requisite for this method is that

  1. You have a Github account.
  1. Your HTML code for this widget is pushed to one of your repositories.
  1. That repository is public.

I have pre-made a “widget” repository that only has notion widget codes which I use solely for this purpose. So the next step is to push this code to the widget repository.

To host the Page

For this step, your GitHub repository has to be modified so that it hosts all the directories within it.

Navigate to the “pages” section of your repository by going to Settings > Pages. Adjust your settings such that Source is set to “Deploy from a branch” and the Branch is set to “main” and “/root”. Now, anything that is pushed to the repository after these changes will be part of your hosted site. The path to the site will be the site path followed by the path to the file as indicated by your directory structure within the repository.

Follow Github’s quick-start guide to Github Pages here: https://docs.github.com/en/pages/quickstart

For me the path to my widget is as follows

<base path to site given by Github>/<repository name>/digitalClock/digitalClock.html

So now I can use this path to embed the widget to my notion page just like any other embedding.

Method 2: Using apps.simple.ink

This website is also known as Notion’s app store. Here you can use other people’s widgets as well as make your own. The caveat is that

  1. Your code cannot be longer than 1500 characters (in my case I have 2414 characters after removing all comments and unnecessary lines). This greatly limits the amount of things you can do in the widget.
  1. You have to restructure your code so that the HTML, CSS and Javascript portions are within one file which may not match how you initially wrote out your code. (Like in my case I wrote them in three different files).

However, if you want to use apps.simple.ink as a way to keep a collection of your widgets anyway, you can use an Iframe which has your widget’s existing hosted link.

Putting your custom widget on apps.simple.ink

  1. Go to apps.simple.ink and make an account.
  1. Select Custom Widgets
  1. Click on Create new widget
  1. Add your iframe link in the HTML code part

If your HTML code is within 1500 characters, you may directly paste the code instead of using the iframe. This website is more useful in those cases.

Now you have an embed link provided to use for your widget

Again, this method is more useful if you want to make a widget which is not hosted already.
